1Signing in & roles

Open the app and you land on the sign-in page. Pick your account, enter your password, and press Sign in.

There are two roles, and they decide what you can do everywhere in the app:

  • Manager  Full control — creates and deletes projects, reassigns Developers, maintains dropdown lists, sees the audit feed, and can delete handover plans.
  • Employee  Sees every project, but can only edit the projects where they are the assigned Developer.
💡Your name and role are always shown in the avatar at the top-right of every page. Use the power button next to it to sign out.

3Creating a new project Manager

Home→ Create button→ Fill General Info→ Save draft / Create
  1. Go to Home (Project Portal) and press Create above the project list.
  2. The Project ID (e.g. PRJ-052) is assigned automatically — the next free number is filled in for you. Just give the project a clear name.
  3. Fill in the basics: work category, business domain, department, Developer (the person responsible for delivery), business owner, and the planned dates.
  4. Press Create at the bottom to activate it. Until then it is saved as a draft — you can leave and come back without losing anything.
💡The Developer matters: it decides which employee is allowed to edit this project. Only Managers can change the Developer later.
⚠️Employees cannot create projects. If the Create button is refused, ask a Manager to create the project and assign you as Developer.

4Filling in project details

Open any project from the Home list to reach its detail page. The page is organised into six scored sections — filling them in is the day-to-day work of the assigned Developer:

SectionWhat goes in it
General InfoIDs, owners, category, domain, dates.
Business InputRequirements and input received from the business side.
E2E ReadinessThe seven delivery stages and their statuses (see next section).
Solution DetailsTechnical design, systems involved, interfaces.
Go-Live ChecklistEverything that must be true before go-live.
Testing / QATest evidence, results, and sign-offs.

Each section contributes to the project's completeness score. The colour bands used everywhere are: < 40% red 40–79% amber ≥ 80% green

💡Edits are saved as a draft first — press Save to publish your changes. Every change is recorded in the audit log.

5Tracking the journey

The Project Journey page turns the E2E Readiness section into a visual road map of seven stages, from kickoff to go-live. Pick a project at the top and the road shows each stage coloured by its status:

Completed In Progress Partially Defined Blocked Missing Input / Not Started Not Applicable

The Go-Live gate meter at the end shows how close the project is to being ready for production. Statuses themselves are changed on the project's detail page (section 4) — the journey page is the read-friendly view you share in meetings.

6Watching data quality

Portfolio Health answers one question: where is information missing?

  • The gauge shows the average completeness of the whole portfolio.
  • Which sections are emptiest? ranks the six sections across all projects — the top row is where the team should focus.
  • Projects needing the most work lists the least-complete projects first. Click a project name to jump straight to its detail page and fix the gaps.

7Executive Dashboard

The Executive Dashboard is the leadership view of the whole portfolio:

  • KPI tiles — project counts, average readiness, risks, and go-live gate status.
  • Readiness trend — a daily snapshot line showing whether the portfolio is improving.
  • Risk register — all projects flagged with risks, worst first.
  • Workload — how many projects each developer carries.
  • Audit feed Manager — the most recent changes across all projects: who changed what, and when.

8Handover to support

When delivery is nearly done, use the Handover page to organise the transfer to the support team.

Pick project→ Create handover plan→ Work the 6 phases→ Mark complete
  1. Pick the project at the top right. Projects that already have a plan show a ✓.
  2. Press Create handover plan. The plan starts with 15 standard tasks across six phases: Preparation → Documentation → Knowledge Transfer → Access & Setup → Hypercare → Sign-off.
  3. Set the plan's target date, who is handing over, and who is receiving.
  4. Work through the tasks: assign an owner, due date, and status to each. Click a task title to rename it, the ✎ button to add notes, and + Task under any phase to add your own tasks. Overdue tasks are flagged automatically.
  5. When every task is done, a green ✓ Mark handover complete button appears — one click closes the plan and stamps the actual handover date.
💡The Current tasks table always shows what to work on next, in journey order. The stepper above it shows how far each phase has progressed.
⚠️Deleting a plan (Manager only) removes the plan and all its tasks. The button asks for a second click to confirm.

9Managing dropdown lists Manager

The values available in dropdowns — work categories, business domains, departments, risk levels, stage statuses, employees — are maintained in Manage Lists.

  • Add a value and it immediately appears in the matching dropdown across the app.
  • Rename a value to correct spelling — existing projects keep their selection.
  • Delete is blocked while any project still uses the value.
💡Add new team members to the Employees list first — otherwise they cannot be selected as Developer or task owner anywhere.

11Tips & troubleshooting

  • "I can't edit a project." You can only edit projects where you are the assigned Developer. Ask a Manager to reassign it if it should be yours.
  • "The Create button doesn't work for me." Only Managers can create projects (section 3).
  • "A dropdown is missing a value I need." A Manager can add it in Manage Lists (section 9) — it appears immediately.
  • "My changes aren't showing for others." Project edits are drafts until you press Save on the detail page.
  • "I was signed out unexpectedly." Sessions end when the browser tab closes. Just sign in again — no work is lost, drafts are kept.
  • Sharing a handover: the Handover page URL includes the project (e.g. ?project=PRJ-001) — copy the address bar link and send it to a colleague.
